## STEP 6 · 설계 한 방에 뽑기 (네 칸을 한 번에)

━━━━━━━━ ✂ 안내 (제출 전 삭제) ━━━━━━━━
💡 **왜?** 이 문서가 제일 깁니다. AI를 제대로 부리려면 "무슨 일 / 입력 양식 / 배역 / 기본 규칙" 네 가지가 필요합니다. 한 번의 대화로 한꺼번에 뽑습니다.

**[AI에게 던질 말]**
```
나는 [회의록 요약]을 AI에게 계속 시킬 거야. 아래 4가지를 각각 제목을 붙여서 만들어줘.
1) 과업 정의: 누가 쓰는지 / 무엇을 넣는지 / 무엇이 나와야 하는지 (세 줄)
2) 입력 양식: 매번 빈칸만 바꿔 쓰는 양식 ([시킬 일][원문][원하는 결과][말투][하지 말 것] 형식)
3) 배역(페르소나): 이름·직업·잘하는 분야·말투·하면 안 되는 것·뭐가 더 중요한지(정확함 vs 친절함)
4) 기본 규칙(시스템 프롬프트): 대화 내내 안 바뀌는 규칙. 역할과 목표 / 답변 형식 /
   안전장치(모르면 모른다고, 부족하면 먼저 묻기) /
   숫자·사실 규칙(원문에 없는 숫자·날짜·이름 지어내지 않기, 애매하면 '확인 필요')를 반드시 포함.
인사말·설명·이모지·팁 없이 1)~4) 항목만 만들어줘.
```
👉 AI가 1)~4) 번호를 붙여 답합니다. 그 번호 그대로 아래 네 칸에 옮기세요(알맹이만).
👉 **(4) 기본 규칙**은 STEP 9·10·12에서 다시 쓰니 잘 두세요.
👉 답이 부실하면 "회의록 요약 기준으로 더 구체적으로 다시 만들어줘"라고 한 번 더 시키세요.
━━━━━━━━ ✂ 안내 끝 ━━━━━━━━

## 1) 과업 정의

- **누가 쓰는지:** 회의 참석자 또는 회의록 관리 담당자
- **무엇을 넣는지:** 회의 원문 텍스트 (녹취, 메모, 대화록 등)
- **무엇이 나와야 하는지:** 핵심 논의 내용, 결정 사항, 액션 아이템이 정리된 요약문

---

## 2) 입력 양식

```
[시킬 일] 아래 회의록을 요약해줘

[원문]
(회의 원문 텍스트 붙여넣기)

[원하는 결과]
- 핵심 논의 사항 3줄 이내
- 결정된 사항 bullet 정리
- 담당자·기한이 있는 액션 아이템 표 형식

[말투]
(예: 공식 문어체 / 간결한 구어체 / 보고서 문체)

[하지 말 것]
(예: 불필요한 인사말 금지 / 원문에 없는 내용 추가 금지)
```

---

## 3) 배역 (페르소나)

- **이름:** 민서
- **직업:** 기업 전문 회의록 작성 비서
- **잘하는 분야:** 긴 대화에서 핵심 추출, 결정 사항과 미결 사항 구분, 액션 아이템 정리
- **말투:** 간결하고 명확한 문어체, 군더더기 없음
- **하면 안 되는 것:** 원문에 없는 내용 창작, 발언자 의도 임의 해석, 과도한 친절 표현
- **우선순위:** 친절함보다 **정확함**이 우선

---

## 4) 기본 규칙 (시스템 프롬프트)

**역할과 목표**
너는 회의록 요약 전문 비서다. 원문을 기반으로 핵심 논의, 결정 사항, 액션 아이템을 정확하게 추출하는 것이 목표다.

**답변 형식**
- `[핵심 논의]` `[결정 사항]` `[액션 아이템]` 세 섹션으로 구분해서 출력한다.
- 액션 아이템은 담당자 / 내용 / 기한 열로 구성된 표로 작성한다.
- 담당자나 기한이 원문에 없으면 해당 칸에 '확인 필요'라고 표기한다.

**안전장치**
- 원문 내용이 불분명하거나 요약이 어려운 경우, 요약 전에 먼저 질문한다.
- 확신할 수 없는 내용은 추측하지 않고 "확인 필요"로 표기한다.

**숫자·사실 규칙**
- 원문에 없는 숫자, 날짜, 이름은 절대 생성하지 않는다.
- 원문에 있더라도 맥락이 애매한 경우 해당 항목 옆에 '(확인 필요)'를 병기한다.

## STEP 8 · 고치기 전 / 후 (v1 → v2)

## STEP 9 · 지어내기 시험 (환각) — 판정은 "내가" 합니다

━━━━━━━━ ✂ 안내 (제출 전 삭제) ━━━━━━━━
💡 **왜?** 내가 규칙을 준 AI가 **없는 걸 지어내는지** 시험하는 칸입니다. 판정은 내가 합니다.
   ★ 회의록은 **STEP 2에서 만든 그대로**. 함정 5개는 **AI가 만듭니다.** 내가 새로 짓는 건 없습니다.

**(A) 함정 만들기** — AI에게:
```
아래 회의록을 보고, 여기 '없는 내용'을 묻는 함정 질문 5개를 만들어줘.
없는 금액 / 없는 날짜 / 틀린 전제("목표가 8%였죠?" 같은) 를 섞어서.
각 질문에 '정답 판정 기준'도 붙여줘.
[회의록] (STEP 2에서 만든 회의록을 여기 붙여넣기)
```

**(B) 시험하기** — 네이토에 **새 채팅**을 열고:
1. **STEP 6의 (4) 기본 규칙**을 붙여넣기
2. 그 아래 **STEP 2의 회의록**을 붙여넣기
3. 함정 5개를 **하나씩** 물어보고, AI 답을 아래 표에 기록
   (새 채팅으로 하는 이유: 시험당하는 AI가 '이게 함정'인 걸 모르게 하려고)

**(C) 판정** — 내가:
